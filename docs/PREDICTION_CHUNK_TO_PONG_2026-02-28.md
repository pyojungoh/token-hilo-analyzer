# 예측픽 chunk_to_pong 가산 완화 (2026-02-28)

## 배경

`graph_analysis_2026-02-28 (1).csv` 연패 구간 분석 결과:

- **chunk_to_pong** 모양에서 연패가 가장 많이 발생 (126회차)
- 13연패 사례: chunk_to_pong 5연속 구간에서 "바뀜" 예측(꺽)을 했는데 실제는 "줄"(정정정정정)이 이어짐
- 덩어리→퐁당 전환 시 과도한 "바뀜" 예측 → 실제로 줄이 이어질 때 연패 다발

## 변경 내용 (쉬운 설명)

### 바꾼 것

| 위치 | 변경 전 | 변경 후 | 의미 |
|------|---------|---------|------|
| **서버** `_apply_phase_line_pong_adjustments` | `pong_w += 0.12` | `pong_w += 0.08` | "덩어리→퐁당"일 때 "바뀜" 예측을 덜 강하게 |
| **클라이언트** `detectVPattern` | `pongW += 0.12`, `lineW -= 0.06` | `pongW += 0.08`, `lineW -= 0.04` | 서버와 같은 비율로 완화 |

### 왜 바꿨나

- **예전**: "덩어리→퐁당"이면 "이제 바뀔 거야"라고 0.12만큼 강하게 예측
- **문제**: 실제로는 줄이 더 이어지는 경우가 많아서, 바뀜 예측이 자주 틀림 → 연패 증가
- **지금**: 0.08로 낮춰서 "바뀔 수도 있고, 줄이 이어질 수도 있다" 정도로만 반영

### 한 줄 요약

> **"덩어리→퐁당" 구간에서 "바뀜" 예측을 덜 과하게 해서, 실제로 줄이 이어질 때 연패를 줄이려는 수정**
